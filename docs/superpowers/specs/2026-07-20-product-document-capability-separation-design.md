# 消保合规产品文档树与能力树分离设计

## 1. 背景

当前产品文档功能直接从 `product_module` 和 `product_feature` 生成 Outline 目录与功能 Spec。因此，文档分类会进入产品模块统计，文档标题也会被误认为标准功能。

消保合规产品需要两套语义独立的结构：

- **产品文档树**：用于存放产品研发、设计、运营和复盘资料。
- **产品能力树**：用于描述产品实际提供的业务模块、子模块和原子功能。

二者可以建立引用关系，但不得互相生成或共享父子层级。

## 2. 目标与非目标

### 2.1 目标

- 在 Outline 中保留 11 个独立文档板块及其子目录、文档正文和历史修订。
- 产品中心“模块与功能”只展示真实业务能力。
- 产品中心“产品文档”读取独立文档树，不再从模块与功能推导目录。
- 功能可选关联一份设计 Spec，但创建或移动功能不会自动改变文档目录。
- 安全迁移当前消保合规数据，不丢失已创建的 Outline 文档。

### 2.2 非目标

- 不改变项目交付文档、商机文档和知识库现有模型。
- 不把 Outline 替换为数据库正文存储。
- 不在本次引入通用文件管理器、全文检索或批量权限继承。
- 不自动删除无法识别来源的 Outline 文档。

## 3. 核心模型

### 3.1 产品文档节点

新增 `product_document_node`：

| 字段 | 说明 |
|---|---|
| `id` | 主键 |
| `product_id` | 所属产品 |
| `parent_id` | 父文档节点，可为空 |
| `node_type` | `FOLDER` 或 `DOCUMENT` |
| `code` | 产品内稳定编码 |
| `title` | 展示标题缓存 |
| `description` | 目录或文档用途说明 |
| `sort_order` | 同级排序 |
| `outline_link_id` | 对应 Outline 索引文档或正文文档 |
| `linked_feature_id` | 可选关联产品功能，仅 `DOCUMENT` 使用 |
| `created_at` / `updated_at` / `version` | 审计及乐观锁 |

约束：

- `(product_id, code)` 唯一。
- `outline_link_id` 唯一。
- `linked_feature_id` 唯一且可为空。
- 父节点必须属于同一产品，树最多四级且不能成环。
- `DOCUMENT` 必须具有 Outline 映射；`FOLDER` 使用 Outline 索引文档映射。
- 删除有子节点的目录前必须先迁移或删除子节点。

Outline 继续保存标题、Markdown 正文和修订历史；MySQL 保存产品关系、目录顺序、同步状态和可选功能关联。

### 3.2 产品能力树

现有 `product_module` 和 `product_feature` 继续作为能力树：

- 一级模块：产品业务域。
- 二级模块：业务子能力。
- 功能：可被版本纳入、需求覆盖和交付复用的原子能力。

从 `ProductStructureController` 移除模块、功能保存后的 Outline 目录同步。产品结构变化只更新业务结构。

### 3.3 功能与 Spec 的关系

- 功能可以没有 Spec。
- 用户可在功能详情中选择“创建 Spec”或“关联已有文档”。
- Spec 是 `product_document_node` 中的普通 `DOCUMENT`，通过 `linked_feature_id` 关联功能。
- 兼容接口 `/products/{productId}/features/{featureId}/spec` 改为查找关联文档节点。
- 未关联 Spec 时返回明确的 `404`，不得隐式创建目录。

## 4. 文档工作区结构

消保合规产品保留以下 11 个一级目录：

```text
01 产品总纲
├── 产品一页纸
├── 产品路线图
├── 产品功能蓝图
└── 方法论词汇表

02 领域建模
├── 领域故事
├── 领域模型图
└── 领域词汇表

03 UI 模式库
├── 模式索引
├── Design Token
└── 交互资产

04 MVP 范围
├── MVP 目标与成功指标
├── MVP 范围清单
├── 非目标与约束
└── MVP 验收基线

05 Spec 工作区
├── Sprint 00 基础与安全
├── Sprint 01 规则中心
├── Sprint 02 审查闭环
├── Sprint 03 消保专项
└── Sprint 04 洞察与运营

06 产品模块库
├── 模块总览
├── 子模块清单
├── 功能清单
└── 依赖与边界

07 ROI 追踪
08 重构工作区
09 学习成长
10 需求收件箱
11 运营日志
```

Sprint 目录下可放置关联功能的设计 Spec。文档位于哪个 Sprint 由产品规划决定，不由功能所属模块决定。

## 6. API 与界面

### 6.1 文档树 API

- `GET /api/v1/products/{productId}/document-nodes`
- `POST /api/v1/products/{productId}/document-nodes`
- `PUT /api/v1/products/{productId}/document-nodes/{nodeId}`
- `POST /api/v1/products/{productId}/document-nodes/{nodeId}/retry`
- `GET /api/v1/products/{productId}/document-nodes/{nodeId}/content`
- `PUT /api/v1/products/{productId}/document-nodes/{nodeId}/content`
- `GET /api/v1/products/{productId}/document-nodes/{nodeId}/export`

删除接口暂不开放，避免误删 Outline 文档；首期通过停用或迁移处理不再使用的节点。

### 6.2 产品文档页

- 左侧展示独立文档树，不显示产品模块编码或功能状态。
- 文件夹仅用于展开、排序和新建子项。
- 文档节点提供预览、编辑、Outline 打开和导出。
- 关联功能时展示功能编码，但不改变目录位置。
- 同步失败只影响对应节点，不阻断产品结构维护。

### 6.3 模块与功能页

- 只展示业务模块、子模块和原子功能。
- 功能详情可查看或创建关联 Spec。
- 模块统计和功能统计不包含任何文档节点。

## 7. 迁移方案

迁移分为四步，任何删除都在验证之后执行。

1. **建立新表并复制文档关系**
   - 将 `DOC-*` 模块迁移为文档文件夹节点。
   - 将 `SPRINT-*` 模块迁移为 Sprint 文件夹节点。
   - 将 `DOC-*` 和 `SPRINT-*-PLAN` 功能迁移为文档节点。
   - 复用现有 `outline_link_id`，不重新创建 Outline 文档。

2. **建立正确能力树**
   - 将 `CAP-*` 提升为一级业务模块。
   - 将当前 31 个粗粒度能力功能转为二级模块。
   - 按第 5 节创建原子功能。
   - 原粗粒度功能的 Outline Spec 迁移为模块说明文档或关联 Spec，不删除正文。

3. **切换读取路径并验证**
   - 产品文档页切换到 `product_document_node`。
   - 模块与功能页只读取能力树。
   - 比对 Outline 文档 ID、标题、父子关系、修订版本和正文摘要。

4. **清理错误业务记录**
   - 解除旧 `product_feature.outline_link_id` 关联。
   - 删除仅用于文档分类的 `DOC-*`、`SPRINT-*` 功能和模块记录。
   - 保留 Outline 文档和 `outline_document_link`。
   - 清理前输出迁移清单；迁移失败时停止，不做部分删除。

迁移任务必须幂等。重复执行只能复用已存在节点，不得生成重复 Outline 文档或重复功能。

## 8. 错误处理与限流

- Outline 创建、读取和移动失败时保存节点及失败状态，业务结构提交不回滚。
- `429` 使用指数退避和抖动重试，批量迁移按节点节流，不调用无间隔的全量同步。
- 版本冲突时提示刷新合并，不静默覆盖 Outline 新修订。
- 迁移中任一文档关系无法证明一致时停止清理阶段并输出问题节点。

## 9. 测试与验收

### 9.1 后端测试

- 文档树父子关系、跨产品父节点、循环和层级上限校验。
- 文档节点编码、Outline 映射和功能关联唯一约束。
- 模块或功能保存不再触发 Outline 目录创建、移动或改名。
- 功能 Spec 创建、关联、读取和版本冲突。
- 迁移幂等、失败中止和 Outline `429` 退避。

### 9.2 前端测试

- 产品文档页展示独立的 11 个目录。
- 模块与功能页不出现 `DOC-*` 或 `SPRINT-*`。
- 功能关联 Spec 后可预览、编辑和导出。
- 文档移动不会改变功能所属模块。

### 9.3 数据验收

- 消保合规产品显示 10 个一级业务模块、31 个二级子模块及完整原子功能。
- Outline 保留 11 个一级文档目录，现有 39 份工作区文档正文和修订不丢失。
- 产品模块数、功能数只统计业务能力。
- 文档树和能力树不存在共享父子关系。
- 所有迁移节点同步状态为 `READY`，失败数为 0。

## 10. 实施顺序

1. 新增独立文档节点表及后端服务。
2. 将产品文档 API 和界面切换为独立树。
3. 移除产品结构保存时的 Outline 自动同步。
4. 增加功能与 Spec 的显式关联。
5. 运行消保合规迁移，验证后清理错误结构。
6. 执行后端、前端和数据验收。
