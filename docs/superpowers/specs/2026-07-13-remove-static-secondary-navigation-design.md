# 移除静态二级导航设计

## 目标

移除所有无法跳转的静态二级菜单及其整列白色侧栏，只保留可点击的一级模块导航，避免界面向用户暗示尚未实现的功能入口。

## 设计

- `AppShell` 改为“一级模块导航 + 工作区”的两列布局。
- 删除 `ModuleNav` 中仅服务于静态二级导航的 `short` 和 `menus` 数据。
- 删除二级侧栏、菜单按钮、版本功能数量文案以及相关图标引用。
- 顶部栏仅显示当前一级模块名称，不再显示虚假的二级路径。
- 保留现有权限过滤、一级导航路由、用户菜单和页面内容渲染逻辑。

## 样式

- 桌面布局从三列改为两列，工作区自动占满释放出的宽度。
- 删除只属于二级侧栏的样式规则。
- 同步调整现有窄屏媒体查询中的网格列定义。

## 验证

- 组件测试确认二级菜单和二级侧栏不再渲染。
- 组件测试继续确认有权限的一级入口可见、无权限入口不可见、页面内容正常显示。
- 运行前端测试与构建，确认类型检查和样式打包通过。
- 刷新当前本地预览，确认一级导航可用且内容区占满剩余空间。

## 非目标

- 不新增二级路由。
- 不改变一级模块、权限或后端接口。
- 不重构无关页面。
