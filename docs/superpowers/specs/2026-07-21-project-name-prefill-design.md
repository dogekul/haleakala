# 创建项目表单顺序与名称预填设计

## 目标

所有创建项目入口先确定客户，再确定产品和版本，第三行显示可编辑的项目名称。当前三处入口包括项目空间、仪表盘快速创建和商机转实施。

## 交互设计

- 第一行：客户。项目空间和仪表盘使用客户选择器；商机转实施显示由商机锁定的只读客户。
- 第二行：产品和版本，保持现有联动与可绑定状态过滤。
- 第三行：项目名称，保留必填校验和“项目编号由系统自动生成”的提示。
- 当客户、产品或版本发生选择变化时，前端使用当前三项生成默认名称：`客户名称 - 产品名称 版本名称 实施项目`。
- 项目名称仍是普通输入框，用户可以修改；提交时始终采用输入框中的最终值。
- 重新选择客户、产品或版本时，默认名称随最新选择重新计算；信息尚不完整时不生成半成品名称。

## 技术方案

新增一个无状态的前端共享函数，接收客户、产品和版本名称并返回默认项目名称。三个表单继续使用 Ant Design `Form`，在选择器的 `onChange` 中读取当前表单值和查询结果，调用共享函数回填 `name`。不新增依赖，不修改后端 API 或数据库。

## 异常与边界

- 版本列表尚未加载、任一选择为空或 ID 无法匹配时，保持现有项目名称，不写入不完整默认值。
- 切换产品时继续清空版本；只有重新选择版本后才生成包含新产品的完整名称。
- 商机预选产品和版本在查询数据加载完成后生成名称；客户来自商机快照。

## 测试

- 单元测试共享名称规则和缺少字段时不生成名称。
- 组件测试验证三行顺序、选择完成后的默认值、人工修改后的提交值。
- 保留并运行现有产品/版本联动、项目提交和商机转交测试。
