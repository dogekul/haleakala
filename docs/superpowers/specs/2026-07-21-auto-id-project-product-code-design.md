# 项目与产品自动编号设计

## 目标

创建项目和产品时不再要求用户填写编码。数据库生成自增主键后，系统将该 ID 的十进制字符串同时作为现有 `code` 字段的值，因此新记录满足 `code == String.valueOf(id)`。

## 方案

保留 `delivery_project.code` 与 `product.code` 字段及现有响应结构，避免影响列表筛选、资源排期、需求分析、Outline 文档标题等既有消费者。创建事务先写入唯一的临时编码，取得自增 ID 后立即将编码更新为 ID 字符串；事务提交前外部无法观察临时值。

已有项目和产品编码保持不变，只约束本次发布后新建的数据。项目、产品详情和列表仍展示 `code`，新数据展示的就是数字 ID。

## 接口与页面

- `POST /api/v1/projects` 不再要求或使用 `code`。
- 商机转交实施并创建项目时不再提交 `project.code`。
- `POST /api/v1/products` 不再要求或使用 `code`；更新产品也不修改编码。
- 项目列表快速创建、仪表盘快速创建、商机转交创建项目、产品创建/编辑抽屉均移除编码输入框。
- 表单在名称附近提示“编号由系统自动生成”，避免用户误认为遗漏字段。

## 一致性与错误处理

临时编码使用 Java 标准库 UUID，并保留现有唯一约束。插入、编码回写及其余初始化操作处于同一事务；任一步失败则整笔创建回滚。无需新增依赖或数据库迁移。

## 测试

- 后端集成测试断言创建请求无需编码，返回值及数据库中的 `code` 等于自增 ID。
- 商机转交测试断言创建项目无需编码。
- 前端测试断言各创建表单不再出现编码输入，提交负载不含 `code`。
- 运行后端全量测试、前端全量测试与生产构建。
