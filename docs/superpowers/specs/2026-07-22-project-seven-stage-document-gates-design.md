# 七阶段项目文档门禁补全设计

## 背景与目标

项目已经具备七阶段状态流转和项目文档空间，但演示数据没有完整的阶段模板，既有项目无法获得后续新增模板，二开文档不能按项目实际情况生效，进入“项目关闭”阶段后也缺少最终关闸动作。

本次按原始交付范式的六个 Gate 卡点，将其适配到当前七阶段模型，补齐模板、同步、条件门禁和项目关闭闭环。客户管理等其他模块不在本次范围内。

## 方案选择

采用“现有知识模板 + 项目快照 + 轻量条件码”的方案：

- 模板仍在知识库维护，发布后作为项目文档模板。
- 项目创建时保存模板快照，避免模板后续修改影响已启动项目。
- 新增模板由项目负责人主动同步，只补缺失项，不覆盖已编辑文档。
- 模板门禁条件只支持 `ALWAYS` 和 `HAS_CUSTOM_DEV`，不引入通用规则引擎。
- 项目关闭必须经过专用动作，不能通过设置页直接改成“关闭”。

不采用“所有文档固定必需”，因为没有二开的项目会被误拦截；也不采用完整工作流/表达式引擎，因为当前只有一个可由系统可靠判断的条件，复杂度没有收益。

## 七阶段模板目录

### 1. 项目启动（START）

必需：售前交付交接检查清单、项目启动检查清单、项目章程与 Kick-off 纪要。

选用：合同与 SOW 转换记录、干系人沟通计划、项目周报、风险登记册、多厂商协同记录。

### 2. 需求调研（REQUIREMENT）

必需：需求采集与规格说明书、需求基线确认书、需求分类与范围确认表。

选用：需求去重与合并矩阵、需求变更记录。

### 3. 二次开发（CUSTOM_DEV）

始终必需：二开阶段门禁确认单，用于明确“存在二开”或“本项目无二开”。

有二开时必需：二开设计文档、测试与代码审查报告、二开上线检查清单。

选用：安全合规检查单、扩展点使用清单。

### 4. 上线实施（GO_LIVE）

必需：UAT 测试及客户签字报告、上线切换与回退方案、上线执行确认单。

选用：业务规则配置表、数据迁移验证报告、客户培训计划、上线指挥手册、安全合规交付检查单。

### 5. 试运行与移交（TRIAL_HANDOVER）

必需：试运行退出报告、运营移交清单、客户培训完成及考核记录。

选用：常见问题 FAQ、驻场管理记录。

### 6. 标准化沉淀（STANDARDIZATION）

始终必需：项目标准化评估报告、标准化债务处置清单、项目复盘与知识沉淀清单。

有二开时必需：二开成本归因报告。

选用：交付产品飞轮建议。

### 7. 项目关闭（CLOSE）

必需：项目验收报告、项目关闭检查清单、合同回款与费用结算确认。

选用：客户满意度评估、续约与增购评估。

## 数据与业务规则

`document_template_config` 与 `project_document` 增加 `condition_code`：

- `ALWAYS`：按 `requirement` 正常判定。
- `HAS_CUSTOM_DEV`：只有项目存在二开任务时才参加门禁判定。

项目文档接口返回 `conditionCode` 和计算后的 `gateRequired`，前端明确展示“必需”“条件必需（已触发）”或“条件文档（未触发）”。

同步模板时只为项目建立尚不存在的模板快照，并创建缺失的 Outline 文档；已有项目文档、确认状态和内容均不覆盖。同步任务可重复触发，执行保持幂等。

阶段推进继续遵守项目门禁模式：

- `BLOCK`：缺失必需文档时禁止推进或关闭。
- `WARNING`：给出缺失清单，但允许用户确认后继续。

进入 `CLOSE` 阶段后项目状态变为 `CLOSING`。只有“完成并关闭项目”动作会校验关闭阶段文档，并将阶段和项目状态改为 `COMPLETED/CLOSED`。设置页不得直接选择 `CLOSING` 或 `CLOSED`。

## 演示数据与既有项目

以独立的可重复 demo 迁移补齐全部模板及其发布快照。迁移使用固定 ID 和 `INSERT IGNORE`，避免覆盖管理员后续修改；同时为既有项目补缺失快照并重新触发文档初始化任务。生产库只获得字段和业务能力，不自动注入演示模板。

## 错误处理

- 同步时项目不存在、无权限或文档空间不可用，沿用统一 API 错误格式。
- 条件码未知时按 `ALWAYS` 安全处理，避免意外放松门禁。
- 关闭失败返回缺失文档标题，前端直接引导用户进入对应阶段补全。
- 文档异步创建失败保留可重试状态，不删除项目快照。

## 验证范围

- 后端集成测试覆盖条件门禁、模板同步幂等、进入关闭和最终关闭。
- 前端测试覆盖条件标识、同步动作、关闭按钮和错误提示。
- 全量后端与前端测试、构建通过后重建本地 Docker 项目。
- 在本地数据库核对七阶段模板数量、既有项目快照和文档任务，并执行页面验收。
