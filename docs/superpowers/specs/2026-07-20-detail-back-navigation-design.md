# 详情页返回导航按钮设计

## 目标

统一商机详情、项目详情、客户运营详情和产品详情中的返回入口，使其具有明确的导航按钮外观、稳定的点击区域和一致的交互反馈，同时保留现有路由与链接语义。

## 范围

- “返回商机总览”，目标路由 `/customers/opportunities`
- “返回项目列表”，目标路由 `/projects`
- “返回客户运营”，目标路由 `/customers/operations`
- “返回产品中心”，目标路由 `/products`

不调整列表页、卡片或详情页中的业务操作按钮。

## 视觉与交互

- 统一使用一个共享样式类，保持四处外观一致。
- 返回入口高度为 32px，采用浅灰描边、白色背景和圆角。
- 左箭头与文字水平排列，保证清楚表达“返回上一层”。
- 悬停时使用浅蓝背景、蓝色边框和蓝色文字。
- 键盘聚焦时显示清晰的蓝色焦点环。
- 移动端保持不小于 32px 的点击高度，并允许文字正常显示。
- 页面内容与返回入口之间使用统一的垂直间距。

## 实现方式

- 保留 React Router 的 `Link`，不引入新的组件或依赖。
- 四个入口统一增加 `detail-back-link` 类。
- 在全局样式中集中定义按钮外观；删除或兼容原有 `detail-back`、`product-detail-back` 的重复样式。
- 项目详情原有的额外包裹元素保留为布局容器，链接本身使用共享类。

## 验证

- 组件测试验证四个返回链接的文案、目标路由和共享样式类。
- 前端完整测试与生产构建通过。
- 重建本地前端容器后，逐页检查默认、悬停和键盘聚焦状态。
