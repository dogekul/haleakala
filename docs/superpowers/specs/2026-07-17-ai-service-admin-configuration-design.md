# AI 服务管理配置设计

## 1. 背景与目标

当前 AI 客户端只从 `AI_BASE_URL`、`AI_MODEL` 和 `AI_API_KEY` 环境变量读取配置。正在运行的本地实例中三项均为空，因此需求分类、甲方诉求清单和差距分析报告会返回“AI 服务尚未配置”。

本次在“系统管理”增加组织级 AI 服务配置，使管理员可以配置 OpenAI 兼容服务、测试连接并安全保存。保存后立即供当前组织的 AI 业务调用使用，不需要重启后端。

## 2. 范围

本次包含：

- 在“系统管理”新增“AI 服务”页面。
- 配置 OpenAI 兼容服务地址、模型名称和 API Key。
- 按组织保存配置，环境变量作为未保存字段的兼容回退。
- API Key 加密入库，读取接口只返回是否已配置。
- 支持测试草稿配置和验证后保存。
- 保存后动态生效，并统一作用于需求分类和商机文档生成。
- 提供稳定的未配置、认证失败、模型不兼容、超时和服务不可用错误。
- 写入不包含密钥的审计记录。

本次不包含：

- 模型供应商账号开通、充值或密钥申请。
- 多模型路由、自动降级、用量计费和调用统计。
- 流式对话、提示词管理、向量模型和图片模型配置。
- 从其他本地项目自动复制密钥。
- 独立的新数据库表或新增第三方依赖。

## 3. 配置模型

复用现有 `system_setting` 表和 `SettingSecretCipher`，使用以下组织级键：

- `ai.baseUrl`
- `ai.model`
- `ai.apiKey`

解析规则与 Outline 配置一致：

1. 当前组织已保存的字段优先。
2. 未保存字段回退到 `delivery.ai.*` 环境配置。
3. 返回配置来源 `ORGANIZATION`、`MIXED` 或 `ENVIRONMENT`。

API Key 必须以 `encrypted=true` 的密文形式保存。读取接口不得返回明文或密文，只返回 `apiKeyConfigured`。保存请求中的 API Key 为空表示保留当前有效密钥；没有当前密钥时，测试和保存均返回校验错误。

## 4. 后端架构

### 4.1 连接快照

新增不可变 `AiConnection`，包含组织 ID、Base URL、模型、API Key 和配置来源。每次调用都从组织配置服务解析连接快照，避免修改全局可变状态和跨组织串用。

### 4.2 配置服务

新增 `AiConfigurationService`，负责：

- 合并组织设置与环境变量回退。
- 校验并规范化 OpenAI 兼容 Base URL。
- 构建未保存的测试草稿。
- 加密保存 API Key。
- 返回脱敏配置视图。

Base URL 只接受 `http` 或 `https`，必须包含主机，不允许用户名密码、查询参数或片段，并移除末尾 `/`。客户端继续兼容传入服务根地址或以 `/v1` 结尾的地址。

### 4.3 AI 客户端

`OpenAiCompatibleClient` 不再在构造时固定读取环境变量，而是按调用方传入的组织 ID 解析 `AiConnection`。实际请求继续使用 OpenAI 兼容的 `/v1/chat/completions` 和 `json_schema` 响应格式。

现有 `AiClient.completeJson` 增加 `organizationId` 参数。需求分类服务和商机阶段文档服务必须传入当前业务数据所属组织，不能依赖线程级全局状态。

### 4.4 管理接口

在 `/api/v1/admin/ai-service` 下提供：

- `GET /config`：返回 Base URL、模型、`apiKeyConfigured` 和来源。
- `POST /config/test`：使用草稿配置发起最小 JSON Schema 请求，不保存。
- `PUT /config`：先执行同样的远程验证，成功后事务保存并返回脱敏配置。

测试请求使用固定、低开销提示词，要求返回 `{ "status": "ok" }`。只有服务可达、认证有效、模型存在并能按系统所需的 JSON Schema 格式返回时才算成功。这样可以在配置阶段发现“接口可连通但业务生成必然失败”的模型不兼容问题。

## 5. 前端交互与样式

在现有系统管理二级导航中新增“AI 服务”，页面沿用 Ant Design 和当前飞书项目风格：

- 顶部标题、说明和当前状态标签。
- 一张两列响应式配置卡片，窄屏变为单列。
- 服务地址输入框、模型名称输入框和 API Key 密码输入框。
- 已有密钥时显示绿色“已配置”，并提示“留空则保持不变”。
- 次操作“测试连接”，主操作“保存配置”。
- 测试成功显示模型与连接成功信息；保存成功刷新脱敏配置。
- 测试和保存拥有独立加载状态，防止重复提交。

页面不显示历史密钥，也不提供复制密钥、清空密钥或供应商快捷注册入口。

## 6. 数据流

管理员打开页面时，前端读取当前组织的脱敏配置。管理员输入新值后可先测试；后端构造仅存在于当前请求中的连接草稿并调用模型。保存时后端再次验证，验证成功后才写入普通配置和加密 API Key，远程验证失败不会覆盖原配置。

业务调用时，服务根据业务数据的组织 ID 解析有效连接，随后调用 OpenAI 兼容接口。组织刚保存的新配置在下一次业务调用中立即生效，无需刷新进程。

## 7. 错误处理与安全

- 缺少任一有效字段：返回“AI 服务尚未配置”。
- HTTP 401/403：返回“AI 服务认证失败，请检查 API Key”。
- 模型不存在或无权限：返回“AI 模型不可用，请检查模型名称和权限”。
- 不支持 JSON Schema 或返回非法 JSON：返回“当前模型不兼容系统结构化输出要求”。
- 连接超时：返回“AI 服务连接超时”。
- 其他网络或上游错误：返回“AI 服务暂时不可用”。

API Key、Authorization Header、密文和完整上游响应不得进入 HTTP 响应、审计详情或应用日志。配置测试和保存仅允许具备现有系统管理权限的用户访问，并沿用当前管理接口的组织隔离和 CSRF 保护。

## 8. 测试与验收

后端测试覆盖：

- 环境变量回退、组织覆盖和多组织隔离。
- API Key 加密保存、脱敏读取和空值保留。
- URL 校验和未知请求字段拒绝。
- 测试失败不保存、测试成功后保存并即时生效。
- AI 客户端按组织选择连接并生成正确请求。
- 认证、超时、模型不兼容和非法 JSON 的稳定错误映射。
- 需求分类和商机文档生成传递正确的组织 ID。
- 管理接口权限和审计内容不泄露密钥。

前端测试覆盖：

- 页面加载脱敏配置和环境变量来源。
- 已配置密钥提示、留空保留语义和表单校验。
- 测试连接、保存、独立加载状态和错误提示。
- 系统管理导航及页面响应式结构。

验收标准：管理员能够在管理页完成配置和测试；保存后不重启服务即可再次推进相关业务；不同组织使用各自配置；任何读取接口、日志和审计记录均不暴露 API Key。
